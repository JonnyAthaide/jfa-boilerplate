# Components

## Objetivo

Criar elementos reutilizáveis da interface.

---

## Responsabilidades

Representar elementos independentes.

---

## Pode conter

- Button
- Modal
- Card
- Badge
- Alert
- Breadcrumb

---

## Nunca deve conter

- Regras de negócio
- Requisições
- Dependências do projeto

---

## Dependências

Foundation

Design System

---

## Fluxo

Componentes são utilizados pelos Layouts e Features.

---

## Boas práticas

Componentes devem ser independentes.

---

## Anti-patterns

Criar componentes específicos do projeto.
