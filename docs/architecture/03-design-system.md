# Design System

## Objetivo

Centralizar toda a identidade visual do framework.

---

## Responsabilidades

- Colors
- Typography
- Spacing
- Shadows
- Radius
- Motion
- Tokens

---

## Pode conter

- Tokens
- Escalas
- Variáveis visuais

---

## Nunca deve conter

- Componentes
- Layouts
- JavaScript

---

## Dependências

Depende de:

- Foundation

Utilizado por:

- Layout
- Components

---

## Fluxo

Transforma tokens em padrões visuais reutilizáveis.

---

## Boas práticas

Manter todos os valores centralizados.

---

## Anti-patterns

Hardcode de cores.

Duplicação de tokens.
